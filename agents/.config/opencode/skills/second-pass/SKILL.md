## When to use
- After a first review's findings have been addressed, to verify the fixes.

## Procedure - findings first
- For each prior finding: resolved / partially resolved / not addressed (one line each).
- New regressions or issues introduced by the fixes.
- Anything still blocking ship.

## Output
- Findings first. Do not edit files.
