# Explain

Explain the requested code, behavior, or repository structure from repository evidence.

Process:

- Read the smallest relevant files first.
- Search only when file locations or symbols are unclear.
- Explain the actual flow, not an inferred architecture.
- Include file references when useful.
- Keep the explanation practical and scoped to the user's question.
- State uncertainty if evidence is incomplete.

Do not edit files unless the user explicitly asks for a change.
