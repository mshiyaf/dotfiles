## When to use
- Auditing a proposed or implemented UI with a critical designer's eye.

## Procedure - findings first
- User flow: is the happy path clear, and are loading / empty / error states covered?
- Consistency with existing design tokens, spacing, and components.
- Accessibility: keyboard, focus order, contrast, semantics.
- AI-slop risks: generic layouts, filler copy, interchangeable cards, inconsistent patterns.
- Responsiveness and long-content behavior.

## Output
- Findings first, concise. Do not edit files.
