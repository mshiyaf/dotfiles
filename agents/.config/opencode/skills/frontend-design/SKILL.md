## Checklist
- Preserve the existing design system when present.
- Ensure desktop and mobile layouts work.
- Use accessible labels, contrast, focus states, and keyboard behavior.
- Avoid generic AI-looking layouts.
- Make spacing, typography, motion, and hierarchy intentional.
- Keep implementation consistent with the project stack.

## Procedure
- Identify the product purpose, audience, and current visual language before changing UI.
- Choose a clear aesthetic direction instead of default gradients and interchangeable cards.
- Prefer distinctive hierarchy, typography, spacing, and interaction details over decorative clutter.
- Verify responsive states and common empty, loading, error, and long-content states.
