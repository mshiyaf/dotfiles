---
name: release-notes
description: Use when drafting changelogs, release notes, version summaries, migration notes, or user-facing change summaries.
---
## Procedure
- Derive entries from commits, PRs, or diffs.
- Group by Added, Changed, Fixed, Removed, Security.
- Write for users and operators, not only developers.
- Mention breaking changes and migration steps clearly.
- Avoid internal-only details unless relevant to release consumers.

## Style
- Prefer plain language over commit jargon.
- Combine noisy commits into one meaningful entry.
- Separate user-facing changes from internal maintenance.
- Include upgrade notes for config, database, API, or behavior changes.
