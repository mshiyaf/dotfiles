name: documentation
description: Use when writing or updating README files, docs, examples, setup instructions, API notes, or developer-facing explanations.
---
## Procedure
- Base docs on repository evidence.
- Preserve existing tone and structure.
- Keep setup, usage, configuration, and troubleshooting clear.
- Include commands only when verified or clearly marked as examples.
- Avoid documenting behavior that is not implemented.
- Prefer concise examples over long prose.
